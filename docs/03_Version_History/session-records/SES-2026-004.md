## 1. Summary

**New era — v4.0.0.** Built the full Meeting-3 workflow: staff **roles** (manager / designer / project_manager), **project assignment** + management-track routing, the **internal price/sqm + manager-approval** queue (staff-only), the **client proposal** builder (Scope/Stages/Pricing/T&C) with client **approve/reject**, and milestone **delivery rules** (attachments, can't-complete-without-a-file, Mood Board + 2D bundle). Both admin surfaces are **role-scoped**; the client portal gained the proposal card, a spaces summary, and dated milestones. This closes out the deferred Meeting-3 work (Increment 1 shipped the role-independent foundation in v3.3.0).

The build was orchestrated with **agents**: a 4-agent workflow built the four surfaces in parallel against a shared schema/role contract, then a 3-agent adversarial review checked cross-surface contracts, price privacy, and fidelity. The review confirmed the core spine correct (pricing → proposal gate → client decision → hub read-back all wire up; the client page never queries `internal_quotes`) and surfaced **2 HIGH + 5 MEDIUM** issues in the seams, which were fixed before ship.

## 3. Schema changes

Applied to prod (`ijtgwmqrxminwypracei`) as migration **`meeting3_workflow_v4`** / repo `supabase/migrations/0002_meeting3_workflow.sql` (additive & idempotent):
- **admins.role** (`manager`|`designer`|`project_manager`; owner seeded `manager`).
- **projects**: `service`, `track` (`design`|`management`), `designer_id`, `pm_id` (FK → admins).
- **internal_quotes** (staff-only): sqm_total, price_per_sqm, total, status (`pending`|`approved`), requested_by, approved_by, approved_at. **Unique index on `project_id`** (one quote per project — prevents the racing-duplicate the review flagged).
- **proposals**: scope, stages, pricing, terms, status (`draft`|`sent`|`approved`|`rejected`), client_comment, sent_at, decided_at.
- **milestones**: `files` jsonb (deliverables), `bundle` text (groups Mood Board + 2D).
- Realtime added for `internal_quotes` + `proposals`.

## 4. Architectural decisions

- **ADR-0010** — staff roles + internal-vs-client quote separation, enforced client-side (server-side/RLS deferred to Phase 2). See the ADR log.
- Client proposal **approval advances the project** (stage → Mood Board, progress 28) and **seeds the Mood Board + 2D bundle** if the plan is empty — so approval isn't a dead-end.
- **7-stage pipeline kept**; Phase 1 (quoting) runs while stage = Quotation, Phase 2 begins at Mood Board.
